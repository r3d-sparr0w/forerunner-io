Forerunner is an open-source EDR/XDR platform — deploy agents across your fleet in minutes, correlate threats against MITRE ATT&CK, and respond from a single dashboard.

Get started View on GitHub
# Install the Forerunner server (TimescaleDB + Redis + server, all-in-one) curl -sSL https://github.com/r3d-sparr0w/forerunner/releases/latest/download/install.sh \ | sudo bash -- --all-in-one

Supports Ubuntu 20.04+, Debian 11+, RHEL/Rocky 8+. Installs TimescaleDB, Redis, and the server binary, generates TLS certificates, and starts a systemd service on port 443.

# From the Forerunner dashboard: Settings > Agents > Deploy Agent # Copy the generated command (includes your enrollment token and server URL): curl -sSL \ -H "X-Enrollment-Token: <token>" \ "https://your-server/api/v1/install/linux-amd64" | sudo bash

The install command is generated per-platform in the dashboard and bundles your CA certificate, enrollment token, and configuration. Linux agents run as a systemd service.

# From the Forerunner dashboard: Settings > Agents > Deploy Agent curl -sSL \ -H "X-Enrollment-Token: <token>" \ "https://your-server/api/v1/install/darwin-arm64" | bash

macOS agents run as a LaunchDaemon. Apple Silicon and Intel binaries are available. No sudo required for the install command itself.

Everything you need. Nothing you don't.

Built for security teams who want visibility without the complexity of commercial EDR.

Real-time telemetry

Process, file, network, and log events streamed from agents via gRPC. Stored in TimescaleDB for high-throughput queries.

Detection engine

Write Sigma-style rules with MITRE ATT&CK tagging. The engine evaluates every event and creates deduplicated alerts automatically.

Threat intelligence

Track threat actor profiles with MITRE technique mappings. Alerts are automatically enriched with actor attribution when rules fire.

Embedded dashboard

Single binary ships the full React dashboard. No separate web server needed — serve and API from the same port 443 endpoint.

Signed releases

Every binary is signed with an Ed25519 key. The install script verifies signatures before execution — no blind pipe-to-bash.

Distributed or all-in-one

Run everything on one host with --all-in-one, or split DB, Redis, and the server across separate nodes for production scale.

Release artifacts

All binaries are signed. Verify with signing.pub before running.

Verifying release signatures

Every release artifact has a companion .sig file containing a base64-encoded Ed25519 signature. The public key lives at forerunner.offsetsec.io/signing.pub.

# Download the public key curl -sSLO https://forerunner.offsetsec.io/signing.pub # Verify a binary (e.g., the Linux amd64 agent) base64 -d forerunner-agent-linux-amd64-musl.sig > sig.raw openssl pkeyutl -verify -pubin -inkey signing.pub \ -rawin -in forerunner-agent-linux-amd64-musl \ -sigfile sig.raw && echo "Signature valid"

The install scripts fetch signing.pub automatically and verify every artifact before installation. You can also review the key fingerprint in the source repository.
