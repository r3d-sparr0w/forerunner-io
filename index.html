Forerunner

Open Source • v0.2.0

Endpoint detection & response built to scale

Forerunner is an open-source EDR/XDR platform — deploy agents across your fleet in minutes, correlate threats against MITRE ATT&CK, and respond from a single self-hosted dashboard.

Get started View on GitHub 
# All-in-one: TimescaleDB + Redis + server binary
curl -sSL https://github.com/r3d-sparr0w/forerunner/releases/latest/download/install.sh \
  | sudo bash -- --all-in-one

Supports Ubuntu 20.04+, Debian 11+, RHEL/Rocky 8+. Installs dependencies, generates mTLS certificates, and starts a systemd service on port 443. For split deployments, pass --db-url and --redis-url instead of --all-in-one.

# From the dashboard: Settings → Agents → Deploy Agent
# Copy the generated command (includes your CA cert, token, and server URL):
curl -sSL \
  -H "X-Enrollment-Token: <token>" \
  "https://your-server/api/v1/install/linux-amd64" | sudo bash

Linux agents install as a systemd service. Statically linked (musl) — runs on any libc variant. The install script verifies the agent binary signature before execution. You can also verify manually.

# From the dashboard: Settings → Agents → Deploy Agent
curl -sSL \
  -H "X-Enrollment-Token: <token>" \
  "https://your-server/api/v1/install/darwin-arm64" | bash

macOS agents run as a LaunchDaemon. Apple Silicon and Intel binaries are available — the install URL detects your architecture automatically. You can also verify manually (requires OpenSSL 3 via Homebrew).

# Windows agent — run in an elevated PowerShell prompt (Administrator)
# Installs as a Windows Service that starts automatically on boot
Invoke-WebRequest `
  -Uri "https://your-server/api/v1/install/windows-amd64" `
  -Headers @{"X-Enrollment-Token" = "<token>"} `
  -OutFile "$env:TEMP\install-agent.ps1"
& "$env:TEMP\install-agent.ps1"

Windows agents (x86_64) install as a Windows service. Requires PowerShell 5.1+ and an Administrator shell. You can also verify manually.

Capabilities

Everything you need. Nothing you don't.

Built for security teams who want visibility without the complexity of commercial EDR.

Real-time telemetry

Process, file, network, and log events streamed from agents via mTLS gRPC. Stored in TimescaleDB for high-throughput queries.

Detection engine

Write Sigma-style rules with MITRE ATT&CK tagging. The engine evaluates every event and creates deduplicated, enriched alerts automatically.

Threat intelligence

Threat actor profiles with MITRE technique mappings. Alerts are automatically enriched with actor attribution when detection rules fire.

Embedded dashboard

Single binary ships the full React dashboard. No separate web server — events, alerts, and response all from one port 443 endpoint.

Signed releases

Every binary is Ed25519-signed. Install scripts fetch and verify signatures before execution — no blind pipe-to-bash.

Distributed or all-in-one

Run everything on one host with --all-in-one, or split DB, cache, and server across nodes for production scale.

Verifying release signatures

Every release artifact ships with a companion .sig file containing a base64-encoded Ed25519 signature. The signing public key is at forerunner.offsetsec.io/signing.pub and mirrored in the source repository.

# Fetch the public key
curl -sSLO https://forerunner.offsetsec.io/signing.pub

# Verify a binary — example: Linux amd64 agent
base64 -d forerunner-agent-linux-amd64-musl.sig > sig.raw
openssl pkeyutl -verify -pubin -inkey signing.pub \
  -rawin -in forerunner-agent-linux-amd64-musl \
  -sigfile sig.raw && echo "Signature valid"

# macOS note: the system `openssl` is LibreSSL, which does not
# support -rawin. Install OpenSSL 3 via Homebrew instead:
#   brew install openssl@3
#   $(brew --prefix openssl@3)/bin/openssl pkeyutl -verify \
#     -pubin -inkey signing.pub -rawin \
#     -in forerunner-agent-darwin-arm64 \
#     -sigfile sig.raw && echo "Signature valid"

Install scripts fetch signing.pub automatically and verify every artifact before installation. You can also review the key fingerprint in the source repository.
